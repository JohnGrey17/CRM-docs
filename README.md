# 🧠 Загальний опис та ідея

Даний функціонал використовується для якісної роботи менеджерів з клієнтами або компаніями та фіксації всіх покрокових етапів взаємодії — від першого контакту до створення замовлення.

Функціонал дозволяє:

- вести структуровану історію роботи з клієнтом
- планувати роботу менеджера
- фіксувати всі етапи комунікації
- аналізувати ефективність роботи менеджерів

Завдяки структурі **Лід → Дії** можна повністю відстежити історію роботи з клієнтом.

# 📖 Головний словник термінів

## Лід

**Лід** — це батьківський процес, який використовується в рамках одного циклу комунікації з клієнтом або компанією.

Він:

- акумулює всю інформацію про клієнта
- містить інформацію про виконану роботу
- є основним інтерфейсом для планування задач
- дозволяє аналізувати комунікацію

В ідеалі **одна комунікація з клієнтом = один лід**.

---

## Дія

**Дія** — це дочірній процес, який створюється всередині ліда.

Особливості:

- один лід може містити багато дій
- дія відповідає за конкретний етап ліду
- у дії фіксується результат виконаної роботи

Таким чином вся робота з лідом складається з **послідовності дій**.

---

## Етапи

Етапи — це статуси, через які проходять процеси.

Вони показують:

- який етап WorkFlow виконується зараз
- куди далі може рухатись процес
- чи завершена задача

# 🧩 Головна ідея бізнес логіки

Основна логіка системи виглядає наступним чином:

1️⃣ Створюється **Лід**  
2️⃣ У ліді формується задача або запит  
3️⃣ Менеджер виконує роботу через **Дії**

Кожна дія:

- планується
- виконується
- фіксується у хронології

Таким чином **усі кроки роботи з клієнтом фіксуються у системі**.

## Блок інформація про запит

![Блок інформація про запит](https://github.com/JohnGrey17/CRM-docs/blob/main/2%D1%96%D0%BD%D1%84%D0%BE%D1%80%D0%BC%D0%B0%D1%86%D1%96%D1%8F%20%D0%BF%D1%80%D0%BE%20%D0%B7%D0%B0%D0%BF%D0%B8%D1%82.png?raw=true)

Основні поля блоку:

**Назва**  
Інформативна назва ліда (наприклад: *Холодний лід з компанією "Тестова"*).  
Поле можна редагувати у будь-який момент.

**Автор**  
Автоматично заповнюється користувачем, який створив лід.

**Відповідальний**  
Користувач, відповідальний за:

- обробку ліда  
- комунікацію з клієнтом  
- контроль дедлайнів  
- завершення ліда

**Дата створення**  
Фіксує дату створення ліда.  
Поле **не редагується**.

**Запланована комунікація**  
Дата наступного контакту з клієнтом.

**Днів без активності**  
Технічне поле, яке показує кількість днів без змін у ліді.

Якщо кількість днів перевищує встановлений поріг, лід автоматично потрапляє до:

```
Корзини неопрацьованих лідів
```

Повернути його в роботу може лише:

- Team Lead
- або відповідальна особа.

**Пріоритет**

Поле визначає важливість ліда.

```
1 — найнижчий пріоритет  
10 — найвищий пріоритет
```

Поле редагується **Team Lead**.

## Блок загальна інформація про клієнта та компанію

![Блок загальна інформація про клієнта](https://github.com/JohnGrey17/CRM-docs/blob/main/5%D0%B7%D0%B0%D0%B3%D0%B0%D0%BB%D1%8C%D0%BD%D0%B0%20%D1%96%D0%BD%D1%84%D0%BE%D1%80%D0%BC%D0%B0%D1%86%D1%96%D1%8F%20%D0%9A%D0%BB%D1%96%D1%94%D0%BD%D1%82%D0%B0.png?raw=true)

**Клієнт**

У цьому полі вказується контактна особа.  
Поле є клікабельним і відкриває карту клієнта.

**Назва компанії**

Аналогічне поле, яке відкриває **карту компанії**.

Телефон, email, ЄДРПОУ та інші поля підтягуються автоматично з карти компанії.

**Кількість замовлень** 
Данне поле використовується у якості лічильника виконаних замовлень клієнта (ті що перейшли на етап "Виконано" ) 
ВАЖЛИВО ЗАМОВЛЕННЯ КЛІЄНТА ТА КОМПАНІЇ РАХУЮТЬСЯ ОКРЕМО І НАРАЗІ НАЛАШТОВАНО НА ЛІЧИЛЬНИК КОМПАНІЇ

# 👤 Карта клієнта

![Карта клієнта](https://github.com/JohnGrey17/CRM-docs/blob/main/%D0%9A%D0%B0%D1%80%D1%82%D0%B0%20%D0%9A%D0%BB%D1%96%D1%94%D0%BD%D1%82%D1%83.png?raw=true)

Карта клієнта містить всю контактну інформацію.

Основні поля:

- ID клієнта
- групи
- ім'я та прізвище
- телефон
- email
- Telegram
- сайт / соціальні мережі
- бажаний спосіб зв'язку

Також доступні:

- примітки про клієнта
- історія взаємодії
- останні події клієнта

# 🏢 Карта компанії

![Карта компанії](https://github.com/JohnGrey17/CRM-docs/blob/main/%D0%9A%D0%B0%D1%80%D1%82%D0%B0%20%D0%BA%D0%BE%D0%BC%D0%BF%D0%B0%D0%BD%D1%96%D1%97.png?raw=true)

Карта компанії містить:

- назву компанії
- ЄДРПОУ
- контактну особу
- співробітників компанії

### Співробітники компанії

У цьому блоці відображаються всі клієнти, які належать до компанії.

Можна:

- додати нового співробітника
- обрати існуючого

### Юридична інформація

Тут зберігається вся юридична інформація компанії, яка використовується у процесах.
