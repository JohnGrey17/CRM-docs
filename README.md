<a id="general-overview"></a>

# 🧠 Загальний опис та ідея

Даний функціонал використовується для якісної роботи менеджерів з клієнтами або компаніями та фіксації всіх покрокових етапів взаємодії — від першого контакту до створення замовлення.

Функціонал дозволяє:

- вести структуровану історію роботи з клієнтом
- планувати роботу менеджера
- фіксувати всі етапи комунікації
- аналізувати ефективність роботи менеджерів

Завдяки структурі **Лід → Дії** можна повністю відстежити історію роботи з клієнтом.

# 📖 Головний словник термінів

## Лід

**Лід** — це батьківський процес, який використовується в рамках одного циклу комунікації з клієнтом або компанією.

Він:

- акумулює всю інформацію про клієнта
- містить інформацію про виконану роботу
- є основним інтерфейсом для планування задач
- дозволяє аналізувати комунікацію

В ідеалі **одна комунікація з клієнтом = один лід**.

---

## Дія

**Дія** — це дочірній процес, який створюється всередині ліда.

Особливості:

- один лід може містити багато дій
- дія відповідає за конкретний етап ліду
- у дії фіксується результат виконаної роботи

Таким чином вся робота з лідом складається з **послідовності дій**.

---

## Етапи

Етапи — це статуси, через які проходять процеси.

Вони показують:

- який етап WorkFlow виконується зараз
- куди далі може рухатись процес
- чи завершена задача

<a id="business-logic"></a>

# 🧩 Головна ідея бізнес логіки

Основна логіка системи виглядає наступним чином:

1️⃣ Створюється **Лід**  
2️⃣ У ліді формується задача або запит  
3️⃣ Менеджер виконує роботу через **Дії**

Кожна дія:

- планується
- виконується
- фіксується у хронології

Таким чином **усі кроки роботи з клієнтом фіксуються у системі**.

## Блок інформація про запит

![Блок інформація про запит](https://github.com/JohnGrey17/CRM-docs/blob/main/2%D1%96%D0%BD%D1%84%D0%BE%D1%80%D0%BC%D0%B0%D1%86%D1%96%D1%8F%20%D0%BF%D1%80%D0%BE%20%D0%B7%D0%B0%D0%BF%D0%B8%D1%82.png?raw=true)

Основні поля блоку:

**Назва**  
Інформативна назва ліда, наприклад: *Холодний лід з компанією "Тестова"*.  
Поле можна редагувати у будь-який момент.

**Автор**  
Автоматично заповнюється користувачем, який створив лід.

**Відповідальний**  
Користувач, відповідальний за:

- обробку ліда
- комунікацію з клієнтом
- контроль дедлайнів
- завершення ліда

**Дата створення**  
Фіксує дату створення ліда.  
Поле **не редагується**.

**Запланована комунікація**  
Дата наступного контакту з клієнтом.

**Днів без активності**  
Технічне поле, яке показує кількість днів без змін у ліді.

Якщо кількість днів перевищує встановлений поріг, лід автоматично потрапляє до:

```text
Корзини неопрацьованих лідів
```

Повернути його в роботу може лише:

- Team Lead
- або відповідальна особа

**Пріоритет**  
Поле визначає важливість ліда.

```text
1 — найнижчий пріоритет
10 — найвищий пріоритет
```

Поле редагується **Team Lead**.

<a id="lead-client-company-info"></a>

## Блок загальна інформація про клієнта та компанію

![Блок загальна інформація про клієнта](https://github.com/JohnGrey17/CRM-docs/blob/main/5%D0%B7%D0%B0%D0%B3%D0%B0%D0%BB%D1%8C%D0%BD%D0%B0%20%D1%96%D0%BD%D1%84%D0%BE%D1%80%D0%BC%D0%B0%D1%86%D1%96%D1%8F%20%D0%9A%D0%BB%D1%96%D1%94%D0%BD%D1%82%D0%B0.png?raw=true)

**Клієнт**  
У цьому полі вказується контактна особа.  
Поле є клікабельним і відкриває карту клієнта.

**Назва компанії**  
Аналогічне поле, яке відкриває **карту компанії**.

Телефон, email, ЄДРПОУ та інші поля підтягуються автоматично з карти компанії.

**Кількість замовлень**  
Дане поле використовується як лічильник виконаних замовлень клієнта, тобто тих, що перейшли на етап **"Виконано"**.

**Важливо:** замовлення клієнта та компанії рахуються окремо. Наразі налаштовано використання лічильника компанії.

<a id="lead-request-details"></a>

## Блок інформаційний блок про запит

![Блок інформаційний блок про запит](https://github.com/JohnGrey17/CRM-docs/blob/main/6%20%D1%96%D0%BD%D1%84%D0%BE%D1%80%D0%BC%D0%B0%D1%86%D1%96%D0%B9%D0%BD%D0%B8%D0%B9%20%D0%B1%D0%BB%D0%BE%D0%BA%20%D0%BF%D1%80%D0%BE%20%D0%B7%D0%B0%D0%BF%D0%B8%D1%82.png?raw=true)

**Опис запиту**  
У цьому полі вказується головна ідея даного ліда або завдання від Team Lead, якщо лід створює він.

**Планування дій**  
Поле **"Планування дій"** та кнопка **"Запланувати дію"** потрібні для створення підпроцесу **Дія**.

Після натискання кнопки:

- текст із поля **"Планування дій"** переноситься до підпроцесу **Дія**
- у ліді поле автоматично очищується

Якщо поле не заповнене, при натисканні кнопки **"Запланувати дію"** з’явиться помилка-попередження.

**Відгук про зразки**  
Поле використовується на етапі **"Тестування зразків"** і містить фідбек клієнта про тестову продукцію.

Без заповнення цього поля натиснути кнопку **"Зразки відтестовано"** не вийде — система видасть помилку.

<a id="lead-active-employee"></a>

## Блок вибір активного співробітника

![Блок вибір активного співробітника](https://github.com/JohnGrey17/CRM-docs/blob/main/9%20%D0%B2%D0%B8%D0%B1%D1%96%D1%80%20%D0%B0%D0%BA%D1%82%D0%B8%D0%B2%D0%BD%D0%BE%D0%B3%D0%BE%20%D1%81%D0%BF%D1%96%D0%B2%D1%80%D0%BE%D0%B1%D1%96%D1%82%D0%BD%D0%B8%D0%BA%D0%B0.png?raw=true)

Даний блок є технічним і наразі виконує функцію контактно-перевірочного модуля.

Менеджери можуть:

- перевірити, чи є користувач у базі OneBox
- шукати по номеру телефону у форматі `0...` або `+380...`
- створити нового користувача

Наразі цей блок використовується для запобігання створенню дублікатів контактів у системі.

У майбутньому планується доробка, щоб при створенні нового користувача або виборі вже існуючого можна було автоматично підставити його в поле **Клієнт** або **Компанія**.

<a id="action-structure"></a>

# 🛠 Структура дії

![Структура дії](https://github.com/JohnGrey17/CRM-docs/blob/main/%D0%86%D0%BD%D1%82%D0%B5%D1%80%D1%84%D0%B5%D0%B9%D1%81%20%D0%B4%D1%96%D1%8F.png?raw=true)

Підпроцес **Дія** складається з таких блоків.

## Інформаційний блок

Містить такі поля:

**Назва**  
За замовчуванням копіює етап ліда, на якому була створена дія, але може бути змінена вручну.

**Відповідальний**  
Підтягується автоматично з ліда.

**Дедлайн**  
Дата, до якої потрібно виконати дію.

**Дата створення**  
Дата створення дії.

**Дата виконання дії**  
Дата, коли дія перейшла на етап **"Виконано"**.

## Блок файли

Сюди можна прикріпляти файли, які використовувались у процесі виконання дії.

У майбутньому можна реалізувати перенесення цих файлів у головний лід після виконання дії.

## Блок менеджера

**Виконана робота**  
Це поле є обов’язковим для переведення дії на етап **"Дія виконана"**.  
Без його заповнення система видасть помилку.

**План дня**  
Переноситься з головного ліда та є описом завдання для дії.
