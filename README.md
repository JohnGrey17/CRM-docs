# 🧠 Загальний опис та ідея

Даний функціонал використовується для якісної роботи менеджерів з клієнтами або компаніями та фіксації всіх покрокових етапів взаємодії — від першого контакту до створення замовлення.

Функціонал дозволяє:

- вести структуровану історію роботи з клієнтом
- планувати роботу менеджера
- фіксувати всі етапи комунікації
- аналізувати ефективність роботи менеджерів

Завдяки структурі **Лід → Дії** можна повністю відстежити історію роботи з клієнтом.

# 📖 Головний словник термінів

## Лід

**Лід** — це батьківський процес, який використовується в рамках одного циклу комунікації з клієнтом або компанією.

Він:

- акумулює всю інформацію про клієнта
- містить інформацію про виконану роботу
- є основним інтерфейсом для планування задач
- дозволяє аналізувати комунікацію

В ідеалі **одна комунікація з клієнтом = один лід**, що дозволяє бачити повну історію взаємодії.

---

## Дія

**Дія** — це дочірній процес, який створюється всередині ліда.

Особливості:

- один лід може містити багато дій
- дія відповідає за конкретний етап роботи
- у дії фіксується результат виконаної роботи

Таким чином вся робота з лідом складається з **послідовності дій**.

---

## Етапи

Етапи — це статуси, через які проходять процеси.

Етапи можуть бути у:

- лідів
- дій

Вони показують:

- на якому етапі знаходиться процес
- який етап WorkFlow виконується зараз
- чи завершена задача

---

# 🧩 Головна ідея бізнес логіки

Основна логіка системи виглядає наступним чином:

1️⃣ Створюється **Лід**  
2️⃣ У ліді формується задача або запит  
3️⃣ Менеджер виконує роботу через **Дії**

Кожна дія:

- планується
- виконується
- фіксується у хронології

Таким чином **усі кроки роботи з клієнтом фіксуються у системі**.

# 🧾 Структура ліда

Лід містить основну інформацію про роботу з клієнтом.

## Блок "Інформація про запит"

Основні поля:

### Назва

Назва ліда носить інформативний характер.

Приклад:

```
Холодний лід з компанією "Тестова"
```

Поле можна редагувати у будь-який момент.

---

### Автор

Автоматично заповнюється користувачем, який створив лід.

---

### Відповідальний

Відповідальна особа за:

- комунікацію з клієнтом
- контроль дедлайнів
- завершення ліда

---

### Дата створення

Фіксує дату створення ліда.

Поле **не редагується**.

---

### Запланована комунікація

Поле використовується для планування наступного контакту з клієнтом.

---

### Днів без активності

Технічне поле, яке показує кількість днів без змін у ліді.

Змінами вважається:

- редагування полів
- створення дій
- виконання дій

Якщо кількість днів перевищує встановлений поріг, лід потрапляє у:

```
Корзину неопрацьованих лідів
```

Повернути його може лише:

- Team Lead
- або відповідальна особа.

---

### Пріоритет

Поле визначає важливість ліда.

Шкала:

```
1 — найнижчий пріоритет  
10 — найвищий пріоритет
```

Поле заповнюється **Team Lead**.

# ❌ Причини відміни

Даний блок використовується для фіксації причин, через які лід був скасований.

Це дозволяє:

- зберегти історію роботи
- аналізувати причини відміни лідів
- вести статистику роботи менеджерів

---

# 👤 Карта клієнта

Поле **Клієнт** містить контактну особу.

Поле є клікабельним і відкриває **розгорнуту карту клієнта**.

Основні поля:

- ім'я
- прізвище
- контактний телефон
- email
- Telegram
- сайт / соціальні мережі

---

### Бажаний спосіб зв'язку

Вказує, де клієнту зручніше спілкуватись.

---

### Telegram Chat ID

Технічне поле для інтеграції Telegram з OneBox.

---

### Лічильник замовлень

Якщо замовлення переходить у статус:

```
Завершено
```

лічильник збільшується на **+1**.

ВАЖЛИВО:

Лічильники для **клієнта та компанії рахуються окремо**.

---

# 🏢 Карта компанії

Карта компанії містить:

- назву компанії
- ЄДРПОУ
- контактну особу
- співробітників компанії

---

### Співробітники

У блоці співробітників відображаються всі клієнти компанії.

Можна:

- додати нового співробітника
- вибрати існуючого

---

### Останні події компанії

Тут відображаються:

- замовлення
- платежі
- інші події

---

# 📝 Блоки роботи з лідом

### Опис запиту

Загальний опис задачі.

---

### Планування дій

Поле використовується разом із кнопкою:

```
Запланувати дію
```

Після натискання:

- текст переноситься у нову дію
- поле очищується.

---

### Відгуки про зразки

Обов'язкове поле на етапі:

```
Тестування зразків
```

---

### Файли

Використовується для прикріплення:

- документів
- файлів
- матеріалів

---

# 📊 Хронологія виконання дій

У цьому блоці автоматично відображається інформація про виконані дії:

- дата створення дії
- назва дії
- опис виконаної роботи
- дата виконання

Редагувати цей блок вручну **неможливо**.

---

# ⚙️ Блок дій менеджера

У цьому блоці відображається коротка інформація про створені дії.

Можна:

- переглянути дію
- перейти до її редагування
- подивитись дату створення
- подивитись дату виконання
